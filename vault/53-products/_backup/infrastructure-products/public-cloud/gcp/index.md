# GCP

Aptum delivers Google Cloud Platform as a partner through Aptum Portal. The GCP plugin is built and available in the portal but not yet configured for general customer use.

## What it is (and what it is not)

Aptum GCP delivery follows the same model as Azure and AWS: consolidated billing, portal visibility, and managed services on top of GCP subscriptions. The GCP plugin in CloudOps Software is built. Configuration for production customer use is the remaining step.

## What Aptum manages vs. what GCP owns

GCP owns the physical infrastructure, global network, and platform SLAs. Aptum manages the partner relationship, consolidated billing, portal cost visibility, and all managed services purchased as addons on top of customer GCP workloads.

## How it's delivered

Via Aptum Portal once the GCP plugin is configured for production use. Google Cloud Partner Interconnect for dedicated private connectivity to Aptum infrastructure is available via the Hybrid Interconnects addon.

## Technical specifications

GCP compute instances, persistent disks, VPCs, and GKE clusters will be manageable through Aptum Portal once configured. Partner Interconnect integration available via the Hybrid Interconnects addon.

## Pricing model

Passthrough of GCP pricing plus partner margin. Managed services addons priced independently per endpoint per month.

## When this fits (and when it doesn't)

GCP through Aptum fits customers already running GCP workloads who want consolidated billing and managed services on top. Activation effort is limited — the plugin is built and waiting for configuration. Same assessment paths as AWS: Hybrid Cloud Assessment and Well-Architected Review.

## Managed services available on top

Full addon catalog compatible with GCP workloads once the plugin is configured. Same addon options as Azure and AWS.
