# App & Platform Modernization Assessment -- Deliverable Scope

> **SOW-ready language.** Copy the relevant size section into the Statement of Work.

---

## Assessment Description

The App & Platform Modernization Assessment evaluates the customer's current application architecture, deployment practices, and platform infrastructure to identify gaps and opportunities for modernization. The assessment covers container readiness, CI/CD maturity, platform architecture, and delivers a phased modernization roadmap.

---

## Objective

To assess the customer's current application delivery and platform engineering maturity, identify opportunities for modernization (containerization, CI/CD, platform architecture), and deliver a practical roadmap for achieving a modern, scalable, and reliable software delivery platform.

---

## T-Shirt Sizing

### Small ($5K) -- Single App Readiness Check

**Scope**: 1 application, basic container readiness and CI/CD maturity evaluation.

**Deliverables**:
- Application architecture overview
- Container readiness assessment (can this app be containerized? what's required?)
- CI/CD maturity quick-scan (current state, key gaps)
- Recommendations memo (5-8 pages)

**Duration**: 1 week

**Aptum team**: Solution Architect (solo)

**Customer time commitment**: 2-3 hours

---

### Medium ($10K) -- Standard Assessment

**Scope**: 1-3 core applications, single dev team. Full container readiness, CI/CD maturity, and platform architecture review.

**Deliverables**:
- Application architecture review (per application)
- Container/Kubernetes readiness evaluation
- CI/CD maturity assessment across the software delivery lifecycle
- Platform architecture review and recommendations
- Technology stack evaluation and recommendations
- Phased modernization roadmap
- Assessment report (15-25 pages)

**Duration**: 2-3 weeks

**Aptum team**: Solution Architect + 1 specialist (DevOps/Platform Engineer)

**Customer time commitment**: 6-10 hours

---

### Large ($20K) -- Multi-App / Multi-Team

**Scope**: 3-10 applications, multiple dev teams, mix of legacy and modern architecture.

**Deliverables**:
- Everything in Medium, plus:
- Multi-application architecture assessment with dependency mapping
- Cross-team practice analysis (where are teams aligned? where do they diverge?)
- Full platform strategy recommendation (build vs. buy vs. managed)
- CI/CD standardization plan across teams
- Migration path from legacy to containerized architecture
- May include proof-of-concept scope definition
- Executive presentation
- Assessment report (25-40 pages)

**Duration**: 3-5 weeks

**Aptum team**: Solution Architect + 2 specialists (DevOps/Platform + Cloud Architect)

**Customer time commitment**: 12-20 hours

---

### XL ($35K+) -- Enterprise / Bespoke

**Scope**: 10+ applications, enterprise-scale platform design. Custom SOW.

**Deliverables**:
- Everything in Large, plus:
- Enterprise platform blueprint (target architecture for all applications)
- Governance framework for platform standards and best practices
- Developer experience assessment and recommendations
- Full Well-Architected Review style engagement across platform pillars
- Executive transformation roadmap with milestones and KPIs
- Board-ready presentation
- Assessment report (40+ pages)

**Duration**: 6-8 weeks

**Aptum team**: Solution Architect + 2-3 specialists + executive sponsor

**Customer time commitment**: 20-30 hours

---

## Out of Scope

- Application code development or refactoring (follow-on engagement)
- CI/CD pipeline implementation (follow-on engagement)
- Kubernetes cluster deployment or management (follow-on)
- Database migration or optimization
- Security testing (see Security Posture Assessment)
- Infrastructure hardware assessment (see Infrastructure Risk Assessment)

---

## Customer Responsibilities

1. **Access**: Provide access to application repositories, CI/CD tools, deployment configurations, and infrastructure
2. **Architecture context**: Share existing architecture diagrams, tech stack documentation, and deployment procedures
3. **Team availability**: Make development team leads and platform/DevOps engineers available for interviews
4. **Demo access**: Provide access to development, staging, and production environments for review
5. **Point of contact**: Designate a technical lead as primary POC

---

## Assumptions

- Customer will provide access to relevant code repositories and CI/CD configurations
- Assessment evaluates architecture and practices; it does not audit code quality or perform code review
- Technology recommendations are based on industry best practices and the customer's specific context
- Modernization roadmap provides direction; detailed implementation design is a follow-on engagement

---

## Acceptance Criteria

1. All deliverables for the selected size provided
2. Findings presented to engineering leadership and relevant stakeholders
3. Final report delivered in PDF format
4. Modernization roadmap delivered with clear phases and priorities
5. Q&A conducted
