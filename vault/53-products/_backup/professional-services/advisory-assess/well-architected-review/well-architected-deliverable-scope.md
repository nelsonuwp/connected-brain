# Well-Architected Review (AWS/Azure) -- Deliverable Scope

> **SOW-ready language.** Copy the relevant size section into the Statement of Work.

---

## Assessment Description

The Well-Architected Review evaluates the customer's cloud environment (AWS and/or Azure) against the industry-standard Well-Architected Framework pillars: Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, and Sustainability. The review identifies architectural risks, optimization opportunities, and best practice gaps, and delivers a prioritized remediation roadmap.

---

## Objective

To assess the customer's cloud architecture against the six Well-Architected Framework pillars, identify high-risk findings and optimization opportunities, and deliver a prioritized, actionable remediation plan to improve security, reliability, performance, and cost efficiency.

---

## T-Shirt Sizing

### Small ($7.5K) -- Focused Review

**Scope**: Single workload or single pillar deep-dive. Single AWS account or Azure subscription.

**Deliverables**:
- Well-Architected Review focused on selected workload(s) or pillar(s)
- Finding inventory with severity classification (High / Medium / Low)
- Top-10 remediation recommendations with priority and effort estimates
- Well-Architected Report document (per Aptum W-AR template, 10-20 pages)

**Duration**: 1-2 weeks

**Aptum team**: Solution Architect (solo)

**Customer time commitment**: 3-4 hours

---

### Medium ($15K) -- Standard Review

**Scope**: Multi-workload review across all 6 pillars. Single AWS account or Azure subscription (or small multi-account).

**Deliverables**:
- Full Well-Architected Review across all 6 pillars
- Detailed finding inventory with severity, risk, and remediation for each
- Cost optimization analysis with specific savings recommendations
- Architecture documentation (current state diagrams)
- Prioritized remediation roadmap
- Well-Architected Report (per Aptum W-AR template, 20-35 pages)

**Duration**: 2-4 weeks

**Aptum team**: Solution Architect + 1 specialist (Cloud Architect)

**Customer time commitment**: 6-10 hours

---

### Large ($25K) -- Enterprise Review

**Scope**: Enterprise-wide, all pillars, multi-account/subscription, remediation roadmap with phased execution plan.

**Deliverables**:
- Everything in Medium, plus:
- Multi-account / multi-subscription architecture review
- Landing zone and organizational structure assessment
- Detailed remediation roadmap with phased plan, effort estimates, and dependencies
- Current state and target state architecture diagrams
- Executive summary presentation
- Well-Architected Report (per Aptum W-AR template, 35-50 pages)

**Duration**: 4-6 weeks

**Aptum team**: Solution Architect + 2 specialists (Cloud + Security Architect)

**Customer time commitment**: 12-20 hours

---

### XL ($40K+) -- Enterprise Transformation

**Scope**: Multi-account, multi-region, full governance framework. Executive transformation roadmap. Custom SOW.

**Deliverables**:
- Everything in Large, plus:
- Multi-region architecture assessment
- Governance framework design (tagging, policies, guardrails, cost controls)
- FinOps / cloud financial management recommendations
- Security baseline and compliance alignment
- Executive workshop for remediation prioritization
- Board-ready transformation roadmap
- Well-Architected Report (customized, 50+ pages)

**Duration**: 6-8+ weeks

**Aptum team**: Solution Architect + 2-3 specialists + executive sponsor

**Customer time commitment**: 20-35 hours

## Out of Scope

- Implementation of remediation recommendations (follow-on engagement)
- Application code review or refactoring
- Penetration testing (see Security Posture Assessment)
- Non-cloud infrastructure assessment (see Infrastructure Risk or Hybrid Cloud Assessment)
- Ongoing cloud optimization (follow-on managed service)
- Cloud provider contract negotiation

---

## Customer Responsibilities

1. **Cloud access**: Provide read-only access to AWS Console / Azure Portal and relevant services
2. **Billing access**: Provide read-only access to AWS Cost Explorer / Azure Cost Management
3. **Architecture context**: Share existing architecture diagrams, runbooks, and operational documentation
4. **Stakeholder availability**: Make cloud engineering, DevOps, security, and finance stakeholders available
5. **Point of contact**: Designate a cloud engineering lead as primary POC

---

## Assumptions

- Customer will provide read-only access to cloud environments within 5 business days of kickoff
- Review evaluates architecture at the time of assessment; cloud environments change frequently
- Cost optimization recommendations use publicly available pricing; actual savings depend on implementation
- The report follows the Aptum W-AR report template (AWS and Azure variants available)
- Remediation roadmap provides direction; detailed implementation is a follow-on engagement

---

## Acceptance Criteria

1. All deliverables for the selected size provided
2. Well-Architected Report delivered per Aptum template format
3. Findings presentation delivered to designated stakeholders
4. Q&A session conducted
5. Cost optimization analysis delivered with specific, actionable savings recommendations
