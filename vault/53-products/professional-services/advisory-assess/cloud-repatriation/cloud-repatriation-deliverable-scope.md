# Cloud Repatriation Assessment -- Deliverable Scope

> **SOW-ready language.** Copy the relevant size section into the Statement of Work.

---

## Assessment Description

The Cloud Repatriation Assessment provides a data-driven evaluation of the customer's public cloud environment to determine which workloads are candidates for repatriation to private cloud or on-premises infrastructure. The assessment includes cloud spend analysis, workload portability assessment, TCO modeling across multiple scenarios, and a phased repatriation roadmap.

---

## Objective

To analyze the customer's current cloud environment, identify workloads that would benefit from repatriation to private or on-premises infrastructure, model the financial impact of repatriation vs. status quo, and deliver a roadmap for executing the recommended approach.

---

## T-Shirt Sizing

### Small ($5K) -- Quick Feasibility Analysis

**Scope**: Cloud spend under $10K/month. High-level repatriation feasibility for the top 3-5 workloads.

**Deliverables**:
- Cloud spend summary and trend analysis (last 6-12 months)
- Top 5 workload profiles with repatriation feasibility rating
- Basic TCO comparison: current cloud cost vs. estimated repatriation cost
- Feasibility memo with recommendation (go / no-go / investigate further)
- Summary document (5-10 pages)

**Duration**: 1 week

**Aptum team**: Solution Architect (solo)

**Customer time commitment**: 2-3 hours (cloud billing access + brief interview)

---

### Medium ($10K) -- Standard Assessment

**Scope**: 5-15 cloud workloads, single cloud provider, cloud spend $10K-$20K/month.

**Deliverables**:
- Detailed cloud spend analysis by service, region, and workload
- Full workload inventory with portability analysis (portable vs. locked-in)
- TCO modeling across 2-3 scenarios (stay, repatriate selective, repatriate aggressive)
- Target infrastructure recommendations for repatriated workloads
- Risk assessment for repatriation candidates
- Phased repatriation roadmap
- Assessment report (15-25 pages)

**Duration**: 2-3 weeks

**Aptum team**: Solution Architect + 1 specialist (Cloud Architect)

**Customer time commitment**: 5-8 hours

---

### Large ($20K) -- Multi-Cloud / Complex

**Scope**: 15-50 cloud workloads, one or two cloud providers, cloud spend $20K-$100K/month. Includes detailed financial modeling and risk analysis.

**Deliverables**:
- Everything in Medium, plus:
- Multi-provider cloud spend analysis with optimization recommendations
- Detailed dependency mapping across workloads
- Workload-by-workload repatriation decision matrix
- Multi-scenario TCO with sensitivity analysis and growth projections
- Target architecture design for the repatriated environment
- Detailed migration plan with effort estimates per workload
- Risk and operational impact assessment
- Executive summary presentation
- Assessment report (25-40 pages)

**Duration**: 3-5 weeks

**Aptum team**: Solution Architect + 2 specialists (Cloud Architect + Infrastructure Architect)

**Customer time commitment**: 10-15 hours

---

### XL ($35K+) -- Enterprise / Bespoke

**Scope**: Enterprise-scale, 50+ workloads, multi-cloud, cloud spend over $100K/month. Full business case with executive presentation. Custom SOW required.

**Deliverables**:
- Everything in Large, plus:
- Enterprise-wide cloud estate analysis across all providers and regions
- Full financial model with business case and ROI for board/investor presentation
- Governance and operational model recommendations for post-repatriation
- Executive workshop for alignment on strategy and priorities
- Change management considerations
- Phased transformation roadmap with executive milestones
- Board-ready executive presentation
- Assessment report (40+ pages, customized format)

**Duration**: 6-8 weeks

**Aptum team**: Solution Architect + 2-3 specialists + Aptum executive sponsor

**Customer time commitment**: 20-30 hours

---

## Out of Scope

- Execution of migration / repatriation (follow-on engagement)
- Renegotiation of cloud provider contracts (customer's responsibility; assessment provides leverage)
- Procurement of target infrastructure
- Application refactoring or modernization (may be identified as a recommendation)
- Ongoing cloud cost optimization (follow-on managed service)

---

## Customer Responsibilities

1. **Cloud billing access**: Provide read-only access to cloud billing console (AWS Cost Explorer, Azure Cost Management, GCP Billing)
2. **Cloud console access**: Provide read-only access to cloud management consoles for resource inventory
3. **Architecture context**: Share existing architecture diagrams and application documentation
4. **Stakeholder availability**: Make engineering, operations, and finance stakeholders available for interviews
5. **Business context**: Share relevant context about business growth plans, compliance requirements, and contract timelines
6. **Point of contact**: Designate a technical POC and executive sponsor

---

## Assumptions

- Customer will provide cloud billing access within 5 business days of kickoff
- Cost analysis based on historical billing data (minimum 3 months, ideally 12 months)
- TCO models use publicly available pricing; actual negotiated rates may differ
- Repatriation feasibility is a recommendation, not a commitment -- implementation decisions remain with the customer
- If the customer has no existing on-prem/colo environment, the assessment will include target infrastructure design recommendations (may increase scope)

---

## Acceptance Criteria

1. All deliverables for the selected size have been provided
2. Findings presentation delivered to designated stakeholders
3. Final report delivered in PDF format
4. TCO model delivered in editable spreadsheet format (for customer's continued use)
5. Q&A session conducted following findings presentation
