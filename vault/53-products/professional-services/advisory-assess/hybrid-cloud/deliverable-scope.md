# Hybrid Cloud Assessment -- Deliverable Scope

> **SOW-ready language.** Copy the relevant size section into the Statement of Work.

---

## Assessment Description

The Hybrid Cloud Assessment provides a vendor-agnostic, strategic evaluation of the customer's current infrastructure and workloads to determine the optimal mix of public cloud, private cloud, and on-premises environments. The assessment includes workload mapping, TCO modeling, architecture review, and a roadmap for achieving the customer's desired target state.

---

## Objective

To evaluate the customer's current infrastructure environment, map workloads to their best-fit platform (public cloud, private cloud, on-prem, or hybrid), model the total cost of ownership across scenarios, and deliver an actionable roadmap for achieving the optimal infrastructure architecture.

---

## T-Shirt Sizing

### Small ($7.5K) -- Single Workload / Quick Analysis

**Scope**: 1-5 applications, single environment. Quick "should we move this?" analysis for a specific workload or service.

**Deliverables**:
- Discovery session with key stakeholder (1-2 hours)
- Workload profile for in-scope application(s)
- Basic TCO comparison: current state vs. 1-2 alternative scenarios
- Recommendation memo with rationale (5-10 pages)

**Duration**: 1-2 weeks

**Aptum team**: Solution Architect (solo)

**Customer time commitment**: 2-3 hours

---

### Medium ($15K) -- Standard Assessment

**Scope**: 5-20 applications/workloads, 1-2 environments (e.g., on-prem + single cloud provider). May include basic compliance considerations.

**Deliverables**:
- Discovery workshop (half day)
- Full infrastructure and workload inventory with dependency mapping
- TCO and suitability modeling across 2-3 scenarios (stay, migrate, hybrid)
- Basic architecture and compliance review
- Target architecture recommendation
- Phased migration/optimization roadmap
- Assessment report (20-30 pages)

**Duration**: 2-4 weeks

**Aptum team**: Solution Architect + 1 specialist (Cloud or Infrastructure Architect)

**Customer time commitment**: 6-10 hours (workshop + interviews + data access)

---

### Large ($25K) -- Complex / Multi-Environment

**Scope**: 20-50 applications/workloads, 2-3 environments, compliance requirements (HIPAA, SOC 2, PCI, data residency), multiple stakeholder teams.

**Deliverables**:
- Everything in Medium, plus:
- Extended discovery workshops (1-2 days across multiple teams)
- Detailed workload-by-workload suitability analysis
- Multi-scenario TCO modeling with growth projections
- Full architecture and compliance review (data residency, regulatory alignment)
- Detailed target architecture design
- Phased roadmap with dependencies, timeline, and resource requirements
- Executive summary presentation
- Assessment report (30-45 pages)

**Duration**: 4-6 weeks

**Aptum team**: Solution Architect + 2 specialists (Cloud Architect + Network/Security Architect)

**Customer time commitment**: 12-20 hours (workshops, interviews, review sessions)

---

### XL ($40K+) -- Enterprise / Bespoke

**Scope**: 50+ applications, global footprint, multi-cloud, heavy regulatory requirements. Custom SOW required.

**Deliverables**:
- Everything in Large, plus:
- Multi-site, multi-region workload analysis
- Full financial modeling with business case and ROI
- Governance framework recommendations
- Executive workshops for alignment on strategy
- Board-ready executive presentation
- Phased transformation roadmap with executive milestone tracking
- Assessment report (50+ pages, customized format)

**Duration**: 6-8 weeks

**Aptum team**: Solution Architect + 2-3 specialists + Aptum executive sponsor

**Customer time commitment**: 20-40 hours (multiple workshops, executive sessions, review cycles)

---

## Out of Scope

- Implementation or execution of migration (follow-on engagement)
- Application code refactoring or modernization (see Platform Modernization Assessment)
- Penetration testing or security vulnerability assessment (see Security Posture Assessment)
- Procurement of cloud services or infrastructure
- Ongoing cloud cost optimization (follow-on managed service)
- Detailed network redesign (may be identified as a follow-on recommendation)

---

## Customer Responsibilities

1. **Access**: Provide access to all in-scope infrastructure environments, cloud consoles, and billing portals
2. **Data**: Share existing architecture diagrams, application inventories, and cloud billing data
3. **Availability**: Make stakeholders (IT, engineering, finance, compliance) available for workshops and interviews
4. **Cloud billing**: Provide read-only access to AWS Cost Explorer, Azure Cost Management, or equivalent
5. **Point of contact**: Designate a single project sponsor and technical POC

---

## Assumptions

- Customer will provide access to environments and cloud billing data within 5 business days of kickoff
- Workload count is approximate; minor variations (within 20%) will not change the assessment scope or price
- Assessment models costs based on current and projected usage; actual costs may vary based on implementation choices
- Cloud provider pricing is based on publicly available rates at the time of assessment
- The assessment does not include implementation of any recommendations

---

## Acceptance Criteria

1. All deliverables for the selected size have been provided
2. A findings presentation has been delivered to designated customer stakeholders
3. The customer has received the final assessment report in PDF format
4. Q&A session has been conducted following the findings presentation
