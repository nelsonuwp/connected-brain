# 30-initiatives  ·  Execution

One note per active initiative. The bridge between thinking and execution.

**Contains:**
- Problem statement and business reason
- Measurable outcomes (not task lists)
- Constraints (time, budget, capability, dependencies)
- Owner and delegation state
- Links to Jira epic and GitLab repo
- LLM work log (context used, output produced, where it landed)

**Statuses:** thinking → speccing → in-flight → blocked → done

**Rules:**
- Earns a note here when it has: an owner, a measurable outcome, and a timeline
- Before that it lives in 10-thinking
- Delegation briefs live in 70-delegation — link from here, don't duplicate
- Decisions link to 60-decisions — don't store them here
- Completed initiatives: set status done, do not delete
